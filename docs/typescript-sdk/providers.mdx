# Voice Providers

Patter supports two voice AI architectures. Each offers different tradeoffs between latency, voice quality, and customization.

## Engine Mode (default)

Pass an `engine` instance to `phone.agent()`. The engine collapses STT, LLM, and TTS into a single streaming provider — lowest latency, fewest moving parts.

### OpenAI Realtime

```typescript
import { Patter, Twilio, OpenAIRealtime } from "getpatter";

const phone = new Patter({ carrier: new Twilio(), phoneNumber: "+15550001234" });

const agent = phone.agent({
  engine: new OpenAIRealtime({ voice: "alloy" }),           // OPENAI_API_KEY from env
  systemPrompt: "You are a helpful assistant.",
});
```

**Supported voices:** `alloy`, `ash`, `ballad`, `coral`, `echo`, `sage`, `shimmer`, `verse`

### ElevenLabs Conversational AI

Uses an agent configured in the ElevenLabs dashboard.

```typescript
import { Patter, Twilio, ElevenLabsConvAI } from "getpatter";

const phone = new Patter({ carrier: new Twilio(), phoneNumber: "+15550001234" });

const agent = phone.agent({
  engine: new ElevenLabsConvAI({ agentId: "agent_abc123" }),   // ELEVENLABS_API_KEY + ELEVENLABS_AGENT_ID from env
  systemPrompt: "You are a warm and friendly concierge.",
});
```

See [Engines](/typescript-sdk/engines) for the full reference.

## Pipeline Mode

Build a custom voice pipeline by combining a separate STT, LLM, and TTS. Omit `engine` and pass `stt` and `tts` instead. Patter also exposes an `onMessage` hook so you can swap in any LLM you like.

```typescript
import { Patter, Twilio, DeepgramSTT, ElevenLabsTTS } from "getpatter";

const phone = new Patter({ carrier: new Twilio(), phoneNumber: "+15550001234" });

const agent = phone.agent({
  stt: new DeepgramSTT({ endpointingMs: 80 }),              // DEEPGRAM_API_KEY from env
  tts: new ElevenLabsTTS({ voiceId: "rachel" }),            // ELEVENLABS_API_KEY from env
  systemPrompt: "You are a helpful assistant.",
});

await phone.serve({
  agent,
  onMessage: async ({ text }) => {
    // Custom LLM logic — return the text to be spoken
    return await myCustomLLM(text);
  },
});
```

### Requirements

Pipeline mode needs both an STT and a TTS instance. If you rely on env vars only, `phone.agent()` raises a clear error at construction time.

## STT Providers

All STT classes are instantiated directly and read their API key from a matching env var when omitted.

```typescript
import { DeepgramSTT, WhisperSTT, CartesiaSTT, AssemblyAISTT, SonioxSTT } from "getpatter";

new DeepgramSTT({ endpointingMs: 80 });               // DEEPGRAM_API_KEY
new WhisperSTT({ language: "es" });                   // OPENAI_API_KEY
new CartesiaSTT({ language: "en", sampleRate: 16000 }); // CARTESIA_API_KEY
new AssemblyAISTT();                                  // ASSEMBLYAI_API_KEY
new SonioxSTT();                                      // SONIOX_API_KEY
```

The namespaced equivalents live under `getpatter/stt/*` (e.g. `import * as deepgram from "getpatter/stt/deepgram"; new deepgram.STT(...)`). See the dedicated [STT page](/typescript-sdk/stt) for the full parameter tables.

## TTS Providers

```typescript
import { ElevenLabsTTS, OpenAITTS, CartesiaTTS, RimeTTS, LMNTTTS } from "getpatter";

new ElevenLabsTTS({ voiceId: "rachel" });             // ELEVENLABS_API_KEY
new OpenAITTS({ voice: "nova" });                     // OPENAI_API_KEY
new CartesiaTTS({ voice: "f786b574-..." });           // CARTESIA_API_KEY
new RimeTTS({ model: "arcana", speaker: "astra" });   // RIME_API_KEY
new LMNTTTS({ model: "blizzard", voice: "leah" });    // LMNT_API_KEY
```

The namespaced equivalents live under `getpatter/tts/*` (e.g. `import * as elevenlabs from "getpatter/tts/elevenlabs"; new elevenlabs.TTS(...)`). See the dedicated [TTS page](/typescript-sdk/tts) for the full parameter tables.

<Note>
OpenAI TTS returns 24 kHz PCM audio, which the SDK automatically resamples to 16 kHz for telephony.
</Note>

## Provider Comparison

| Feature | `engine: OpenAIRealtime` | `engine: ElevenLabsConvAI` | Pipeline |
|---------|--------------------------|-----------------------------|----------|
| Latency | Lowest | Low | Variable |
| Voice quality | High | Very high | Depends on TTS |
| Custom LLM | No | No | Yes |
| Function calling | Yes | Limited | Via `onMessage` |
| Languages | Multi | Multi | Depends on STT/TTS |
