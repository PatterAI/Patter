# Voice Providers

Patter supports two voice AI architectures. Each offers different tradeoffs between latency, voice quality, and customization.

## Engine Mode (default)

Pass an `engine=` instance to `phone.agent()`. The engine collapses STT, LLM, and TTS into a single streaming provider — lowest latency, fewest moving parts.

### OpenAI Realtime

```python
from getpatter import Patter, Twilio, OpenAIRealtime

phone = Patter(carrier=Twilio(), phone_number="+15550001234")   # TWILIO_* from env

agent = phone.agent(
    engine=OpenAIRealtime(voice="alloy"),                       # OPENAI_API_KEY from env
    system_prompt="You are a helpful assistant.",
)
```

### Available voices

`"alloy"`, `"echo"`, `"fable"`, `"onyx"`, `"nova"`, `"shimmer"`

### Audio Encoding

OpenAI Realtime handles audio encoding automatically based on your telephony provider:

| Telephony Provider | Audio Format | Sample Rate |
|-------------------|--------------|-------------|
| Twilio | G.711 mu-law | 8 kHz |
| Telnyx | PCM 16-bit | 16 kHz |

### ElevenLabs Conversational AI

Uses an agent configured in the ElevenLabs dashboard. Ideal when voice quality is the top priority.

```python
from getpatter import Patter, Twilio, ElevenLabsConvAI

phone = Patter(carrier=Twilio(), phone_number="+15550001234")

agent = phone.agent(
    engine=ElevenLabsConvAI(agent_id="agent_abc123"),           # ELEVENLABS_API_KEY from env
    system_prompt="You are a warm and friendly concierge.",
)
```

See [Engines](/python-sdk/engines) for the full reference.

## Pipeline Mode

Build a custom voice pipeline by combining a separate STT, LLM, and TTS. Omit `engine=` and pass `stt=` and `tts=` instead. Patter also exposes an `on_message` hook so you can swap in any LLM you like.

```python
import asyncio
from getpatter import Patter, Twilio, DeepgramSTT, ElevenLabsTTS

phone = Patter(carrier=Twilio(), phone_number="+15550001234")

agent = phone.agent(
    stt=DeepgramSTT(endpointing_ms=80),                         # DEEPGRAM_API_KEY from env
    tts=ElevenLabsTTS(voice_id="rachel"),                       # ELEVENLABS_API_KEY from env
    system_prompt="You are a helpful assistant.",
)

async def handle_message(event) -> str:
    # Drop in OpenAI, Anthropic, Groq, Cerebras, Ollama, whatever.
    return f"You said: {event['text']}. How can I help?"

async def main():
    await phone.serve(agent, on_message=handle_message)

asyncio.run(main())
```

### Requirements

Pipeline mode needs both an STT and a TTS instance. If you rely on env vars only, `phone.agent()` raises a clear error at construction time.

## STT Providers

All STT classes are instantiated directly and read their API key from a matching env var when omitted.

```python
from getpatter import DeepgramSTT, WhisperSTT, CartesiaSTT, AssemblyAISTT, SonioxSTT

DeepgramSTT(endpointing_ms=80)                # DEEPGRAM_API_KEY
WhisperSTT(language="es")                     # OPENAI_API_KEY
CartesiaSTT(language="en", sample_rate=16000) # CARTESIA_API_KEY
AssemblyAISTT()                               # ASSEMBLYAI_API_KEY
SonioxSTT()                                   # SONIOX_API_KEY
```

The namespaced equivalents live under `getpatter.stt` (e.g. `from getpatter.stt import deepgram; deepgram.STT(...)`). See the dedicated [STT page](/python-sdk/stt) for the full parameter tables.

## TTS Providers

```python
from getpatter import ElevenLabsTTS, OpenAITTS, CartesiaTTS, RimeTTS, LMNTTTS

ElevenLabsTTS(voice_id="rachel")              # ELEVENLABS_API_KEY
OpenAITTS(voice="nova")                       # OPENAI_API_KEY
CartesiaTTS(voice="f786b574-...")             # CARTESIA_API_KEY
RimeTTS(model="arcana", speaker="astra")      # RIME_API_KEY
LMNTTTS(model="blizzard", voice="leah")       # LMNT_API_KEY
```

The namespaced equivalents live under `getpatter.tts` (e.g. `from getpatter.tts import elevenlabs; elevenlabs.TTS(...)`). See the dedicated [TTS page](/python-sdk/tts) for the full parameter tables.

<Note>
  OpenAI TTS returns audio at 24 kHz. Patter automatically resamples it to 16 kHz for telephony compatibility.
</Note>

## Provider Comparison

| Feature | `engine=OpenAIRealtime()` | `engine=ElevenLabsConvAI()` | Pipeline |
|---------|---------------------------|------------------------------|----------|
| Latency | Lowest | Low | Medium |
| Voice quality | Good | Best | Configurable |
| Customization | Limited | Medium | Full |
| `on_message` callback | No | No | Yes |
| Required env vars | `OPENAI_API_KEY` | `ELEVENLABS_API_KEY`, `ELEVENLABS_AGENT_ID` | STT + TTS keys |

## Complete Pipeline Example

```python
import os
import asyncio
from dotenv import load_dotenv
from getpatter import Patter, Twilio, DeepgramSTT, ElevenLabsTTS

load_dotenv()

phone = Patter(
    carrier=Twilio(),                                           # TWILIO_* from env
    phone_number=os.environ["PHONE_NUMBER"],
    webhook_url=os.environ["WEBHOOK_URL"],
)

agent = phone.agent(
    stt=DeepgramSTT(),                                          # DEEPGRAM_API_KEY from env
    tts=ElevenLabsTTS(voice_id="rachel"),                       # ELEVENLABS_API_KEY from env
    system_prompt="You are a helpful assistant.",
)

async def handle_message(event) -> str:
    user_text = event["text"]
    # Add your own LLM logic here
    return f"I heard you say: {user_text}"

async def main():
    await phone.serve(agent, on_message=handle_message, port=8000)

asyncio.run(main())
```
