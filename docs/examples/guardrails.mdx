# Guardrails Example

Guardrails intercept AI responses before they are sent to TTS. When a response matches a blocked term or fails a custom check, a replacement message is spoken instead.

<CodeGroup>
```python Python
import asyncio
from patter import Patter

phone = Patter(
    twilio_sid="ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    twilio_token="your_auth_token",
    openai_key="sk-xxx",
    phone_number="+15550001234",
    webhook_url="abc123.ngrok.io",
)

agent = phone.agent(
    system_prompt="You are a customer service agent for Acme Corp.",
    voice="alloy",
    first_message="Hello! How can I help you today?",
    guardrails=[
        # Block specific terms
        Patter.guardrail(
            name="no_medical_advice",
            blocked_terms=["diagnosis", "prescription", "medication", "dosage"],
            replacement="I'm not qualified to provide medical advice. Please consult a healthcare professional.",
        ),
        # Block competitor mentions
        Patter.guardrail(
            name="no_competitors",
            blocked_terms=["CompetitorCo", "RivalInc"],
            replacement="I can only discuss Acme Corp products and services.",
        ),
        # Custom check function
        Patter.guardrail(
            name="no_pricing_promises",
            check=lambda text: "guarantee" in text.lower() and "price" in text.lower(),
            replacement="I cannot make pricing guarantees. Let me connect you with our sales team.",
        ),
        # Combine blocked terms and custom check
        Patter.guardrail(
            name="content_policy",
            blocked_terms=["profanity_word_1", "profanity_word_2"],
            check=lambda text: len(text) > 500,  # Block overly long responses
            replacement="Let me give you a more concise answer.",
        ),
    ],
)

asyncio.run(phone.serve(agent))
```

```typescript TypeScript
import { Patter } from "patter";

const phone = new Patter({
  mode: "local",
  twilioSid: "ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  twilioToken: "your_auth_token",
  openaiKey: "sk-xxx",
  phoneNumber: "+15550001234",
  webhookUrl: "abc123.ngrok.io",
});

const agent = phone.agent({
  systemPrompt: "You are a customer service agent for Acme Corp.",
  voice: "alloy",
  firstMessage: "Hello! How can I help you today?",
  guardrails: [
    // Block specific terms
    Patter.guardrail({
      name: "no_medical_advice",
      blockedTerms: ["diagnosis", "prescription", "medication", "dosage"],
      replacement:
        "I'm not qualified to provide medical advice. Please consult a healthcare professional.",
    }),
    // Block competitor mentions
    Patter.guardrail({
      name: "no_competitors",
      blockedTerms: ["CompetitorCo", "RivalInc"],
      replacement: "I can only discuss Acme Corp products and services.",
    }),
    // Custom check function
    Patter.guardrail({
      name: "no_pricing_promises",
      check: (text) =>
        text.toLowerCase().includes("guarantee") &&
        text.toLowerCase().includes("price"),
      replacement:
        "I cannot make pricing guarantees. Let me connect you with our sales team.",
    }),
    // Combine blocked terms and custom check
    Patter.guardrail({
      name: "content_policy",
      blockedTerms: ["profanity_word_1", "profanity_word_2"],
      check: (text) => text.length > 500, // Block overly long responses
      replacement: "Let me give you a more concise answer.",
    }),
  ],
});

await phone.serve({ agent });
```
</CodeGroup>

## How Guardrails Work

1. The AI generates a response
2. Each guardrail is evaluated in order:
   - `blocked_terms` / `blockedTerms` are checked first (case-insensitive substring match)
   - `check` function is evaluated next (if provided)
3. If any guardrail triggers, the `replacement` text is spoken instead
4. If no guardrail triggers, the original response is spoken

<Note>
Guardrails are evaluated in the order they are listed. The first guardrail to trigger wins. If multiple guardrails might match, put the most specific ones first.
</Note>

## Use Cases

| Guardrail | Purpose |
|-----------|---------|
| Blocked terms | Prevent the agent from mentioning competitors, medical terms, legal advice |
| Length check | Keep responses concise for phone conversations |
| Regex check | Block responses matching specific patterns (e.g., phone numbers, emails) |
| Sentiment check | Block negative or aggressive responses |
| Compliance check | Ensure financial/legal compliance by blocking regulated language |
