## Overview

Patter uses WebSockets for real-time bidirectional communication. There are two categories of WebSocket connections:

1. **SDK WebSocket** -- connects your application to the Patter backend for receiving events and sending commands.
2. **Stream WebSockets** -- carry audio between telephony providers and the backend. These are managed by Patter and not directly used by your application.

---

## SDK WebSocket

Connects your SDK client to the Patter backend for real-time call events and control.

```
ws://api.patter.ai/ws/sdk
```

### Authentication

Pass your API key via the `X-API-Key` header during the WebSocket handshake.

```python
import websockets

async with websockets.connect(
    "wss://api.patter.ai/ws/sdk",
    extra_headers={"X-API-Key": "pt_your_api_key"}
) as ws:
    async for message in ws:
        print(message)
```

### Connection Limits

| Parameter | Value |
|---|---|
| Max message size | 64 KB |
| Auth | `X-API-Key` header |
| Protocol | `wss://` in production |

### Message Format

All messages are JSON-encoded.

#### Server-to-Client Events

**Call started:**

```json
{
  "event": "call.started",
  "data": {
    "call_id": "call_abc123",
    "agent_id": "agt_xyz789",
    "direction": "inbound",
    "from": "+14155551234",
    "to": "+14155559876",
    "started_at": "2025-01-15T10:30:00Z"
  }
}
```

**Transcript update:**

```json
{
  "event": "call.transcript",
  "data": {
    "call_id": "call_abc123",
    "role": "user",
    "text": "I need help with my billing.",
    "timestamp": "2025-01-15T10:30:08Z"
  }
}
```

**Tool invocation:**

```json
{
  "event": "call.tool_call",
  "data": {
    "call_id": "call_abc123",
    "tool_name": "lookup_order",
    "arguments": {"order_id": "ORD-12345"}
  }
}
```

**Call ended:**

```json
{
  "event": "call.ended",
  "data": {
    "call_id": "call_abc123",
    "duration_seconds": 127,
    "ended_at": "2025-01-15T10:32:07Z"
  }
}
```

**Conversation history** (available in all callback events):

```json
{
  "event": "call.transcript",
  "data": {
    "call_id": "call_abc123",
    "role": "agent",
    "text": "Let me look that up for you.",
    "timestamp": "2025-01-15T10:30:10Z",
    "history": [
      {"role": "agent", "text": "Hello! How can I help?", "timestamp": "2025-01-15T10:30:02Z"},
      {"role": "user", "text": "I need help with my billing.", "timestamp": "2025-01-15T10:30:08Z"},
      {"role": "agent", "text": "Let me look that up for you.", "timestamp": "2025-01-15T10:30:10Z"}
    ]
  }
}
```

#### Client-to-Server Messages

**Tool result:**

```json
{
  "type": "tool_result",
  "data": {
    "call_id": "call_abc123",
    "tool_name": "lookup_order",
    "result": {"status": "shipped", "tracking": "1Z999AA10123456784"}
  }
}
```

**Transfer call:**

```json
{
  "type": "transfer_call",
  "data": {
    "call_id": "call_abc123",
    "to": "+14155550000"
  }
}
```

## Telephony Stream WebSockets

These WebSocket endpoints are used internally by Patter to receive audio streams from telephony providers. They are not called by your application -- the telephony provider connects to them automatically when a call begins.

### Twilio Stream

```
wss://api.patter.ai/ws/stream/twilio/{call_sid}?token={hmac_token}
```

| Parameter | Description |
|---|---|
| `call_sid` | Twilio Call SID |
| `token` | HMAC authentication token (generated by Patter) |

Audio format: mulaw 8kHz (transcoded internally).

### Telnyx Stream

```
wss://api.patter.ai/ws/stream/telnyx/{id}?token={hmac_token}
```

| Parameter | Description |
|---|---|
| `id` | Telnyx connection identifier |
| `token` | HMAC authentication token (generated by Patter) |

Audio format: 16kHz PCM (native, no transcoding needed).

### OpenAI Realtime Bridge (Twilio)

```
wss://api.patter.ai/ws/stream/realtime/twilio/{call_sid}?token={hmac_token}
```

Bridges Twilio audio directly to OpenAI's Realtime API for speech-to-speech processing. Used when the agent is configured for OpenAI Realtime mode.

### ElevenLabs Bridge (Twilio)

```
wss://api.patter.ai/ws/stream/elevenlabs/twilio/{call_sid}?token={hmac_token}
```

Bridges Twilio audio to ElevenLabs for low-latency voice synthesis. Used when the agent's `voice_provider` is set to `elevenlabs`.

## Audio Pipeline

The audio flow through the system:

```
Phone Call
  -> Telephony Provider (Twilio/Telnyx)
  -> Stream WebSocket (audio in)
  -> Backend (transcode if needed)
  -> STT (speech-to-text)
  -> SDK WebSocket (text to your app)
  -> LLM (generate response)
  -> TTS (text-to-speech)
  -> Backend (resample to provider format)
  -> Stream WebSocket (audio out)
  -> Telephony Provider
  -> Phone Call
```

### Barge-In

Patter supports mark-based barge-in for precise interruption handling. When the caller starts speaking while the agent is talking:

1. Audio playback marks track which audio chunks have been delivered
2. The system identifies the exact point of interruption
3. Unplayed audio is discarded
4. The agent receives the new user input immediately

### AI Disclosure

An AI disclosure message plays automatically at the start of every call. This is non-optional and precedes the agent's `first_message`.
