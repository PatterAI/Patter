## Get Usage

Returns the current billing plan, balance, usage limits, and consumption for the billing period.

```bash
curl https://api.patter.ai/api/v1/billing/usage \
  -H "X-API-Key: pt_your_api_key"
```

### Response `200`

```json
{
  "data": {
    "plan": "pay_as_you_go",
    "balance": 45.50,
    "limits": {
      "monthly_minutes": null,
      "concurrent_calls": 10
    },
    "usage": {
      "minutes_used": 342,
      "calls_made": 89,
      "period_start": "2025-01-01T00:00:00Z",
      "period_end": "2025-02-01T00:00:00Z"
    }
  }
}
```

**Rate limit**: 60 requests per minute.

## Add Funds

Adds credits to your account balance. Requires a completed Stripe payment.

```bash
curl -X POST https://api.patter.ai/api/v1/billing/add-funds \
  -H "X-API-Key: pt_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 50.00,
    "stripe_payment_id": "pi_abc123def456"
  }'
```

### Request Body

<ParamField body="amount" type="number" required>
  Amount to add in USD. Minimum `$5.00`, maximum `$1,000.00`.
</ParamField>

<ParamField body="stripe_payment_id" type="string" required>
  Stripe PaymentIntent ID from a completed payment.
</ParamField>

### Response `200`

```json
{
  "data": {
    "balance": 95.50,
    "amount_added": 50.00,
    "transaction_id": "txn_abc123"
  }
}
```

**Rate limit**: 10 requests per minute.

## List Transactions

Returns a paginated list of billing transactions (charges, credits, refunds).

```bash
curl "https://api.patter.ai/api/v1/billing/transactions?limit=20&offset=0" \
  -H "X-API-Key: pt_your_api_key"
```

### Query Parameters

<ParamField query="limit" type="integer" default="20">
  Number of results to return (max 100).
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip.
</ParamField>

### Response `200`

```json
{
  "data": [
    {
      "id": "txn_abc123",
      "type": "credit",
      "amount": 50.00,
      "description": "Added funds",
      "balance_after": 95.50,
      "created_at": "2025-01-15T10:30:00Z"
    },
    {
      "id": "txn_def456",
      "type": "charge",
      "amount": -0.12,
      "description": "Call call_abc123 (2m 7s)",
      "balance_after": 45.50,
      "created_at": "2025-01-15T10:32:07Z"
    }
  ],
  "total": 156,
  "limit": 20,
  "offset": 0
}
```

**Rate limit**: 60 requests per minute.

## Change Plan

Switches the account to a different billing plan.

```bash
curl -X POST https://api.patter.ai/api/v1/billing/plan \
  -H "X-API-Key: pt_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "plan": "pro",
    "stripe_payment_id": "pi_xyz789"
  }'
```

### Request Body

<ParamField body="plan" type="string" required>
  The plan identifier to switch to (e.g., `free`, `pay_as_you_go`, `pro`, `enterprise`).
</ParamField>

<ParamField body="stripe_payment_id" type="string">
  Stripe PaymentIntent or Subscription ID. Required for paid plans.
</ParamField>

### Response `200`

```json
{
  "data": {
    "plan": "pro",
    "effective_at": "2025-01-15T10:30:00Z"
  }
}
```

**Rate limit**: 10 requests per minute.

## Stripe Webhook

Receives Stripe webhook events for payment confirmations, subscription updates, and failures. This endpoint is called by Stripe, not by your application.

```
POST https://api.patter.ai/api/v1/billing/stripe-webhook
```

### Headers

| Header | Description |
|---|---|
| `Stripe-Signature` | Stripe webhook signature for verification |

### Handled Events

| Event | Action |
|---|---|
| `payment_intent.succeeded` | Credits added to account |
| `payment_intent.payment_failed` | Notification sent, no balance change |
| `customer.subscription.updated` | Plan updated |
| `customer.subscription.deleted` | Plan downgraded to free |
| `invoice.payment_failed` | Account flagged, grace period starts |

<Note>
  This endpoint is for Stripe-to-Patter communication. You do not need to call it directly. Configure the webhook URL in your Stripe dashboard.
</Note>
