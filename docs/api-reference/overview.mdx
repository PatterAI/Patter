## Base URL

All API requests are made to:

```
https://api.patter.ai/api/v1/
```

## Authentication

Most endpoints require an API key passed via the `X-API-Key` header. API keys use the `pt_*` prefix and are SHA-256 hashed before storage.

```bash
curl https://api.patter.ai/api/v1/accounts/me \
  -H "X-API-Key: pt_your_api_key_here"
```

See [Authentication](/api-reference/authentication) for details on creating and rotating keys.

## Rate Limiting

All endpoints are rate-limited via `slowapi`. Limits are specified per endpoint and measured per API key. When you exceed the limit, the API returns `429 Too Many Requests`.

| Rate limit | Typical endpoints |
|---|---|
| **5/min** | Key rotation, provisioning, account deletion, plan changes |
| **10/min** | Account creation, number registration, reactivation, add funds |
| **30/min** | Agent CRUD, assign/unassign agent |
| **60/min** | List/read endpoints (agents, calls, numbers, billing) |

Rate limit headers are included in every response:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58
X-RateLimit-Reset: 1714000000
```

## Request Format

- **Content-Type**: `application/json` for all request bodies
- **Character encoding**: UTF-8
- **Date format**: ISO 8601 (`2025-01-15T10:30:00Z`)
- **Phone numbers**: E.164 format (`+14155551234`)

## Response Format

All responses follow a consistent envelope:

```json
{
  "data": { ... },
  "error": null
}
```

On error:

```json
{
  "data": null,
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Human-readable error description"
  }
}
```

## HTTP Status Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `201` | Resource created |
| `204` | Success, no content (e.g., delete) |
| `400` | Bad request / validation error |
| `401` | Missing or invalid API key |
| `403` | Forbidden (insufficient permissions) |
| `404` | Resource not found |
| `409` | Conflict (e.g., duplicate resource) |
| `422` | Unprocessable entity |
| `429` | Rate limit exceeded |
| `500` | Internal server error |

## Error Codes

| Error code | Description |
|---|---|
| `INVALID_REQUEST` | Request body failed validation |
| `UNAUTHORIZED` | API key is missing or invalid |
| `FORBIDDEN` | Key does not have access to this resource |
| `NOT_FOUND` | Requested resource does not exist |
| `CONFLICT` | Resource already exists or state conflict |
| `RATE_LIMITED` | Too many requests, retry after the reset time |
| `INSUFFICIENT_FUNDS` | Account balance too low for the operation |
| `PROVIDER_ERROR` | Upstream telephony provider returned an error |
| `INTERNAL_ERROR` | Unexpected server error |

## Pagination

List endpoints support pagination via query parameters:

| Parameter | Type | Default | Description |
|---|---|---|---|
| `limit` | integer | `20` | Number of items to return (max 100) |
| `offset` | integer | `0` | Number of items to skip |

Paginated responses include metadata:

```json
{
  "data": [...],
  "total": 142,
  "limit": 20,
  "offset": 0
}
```
