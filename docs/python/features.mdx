# Features

## Guardrails

Guardrails intercept AI responses before they reach TTS. When a response is blocked, a replacement message is spoken instead.

```python
agent = phone.agent(
    system_prompt="You are a customer service agent.",
    guardrails=[
        Patter.guardrail(
            name="no_medical",
            blocked_terms=["diagnosis", "prescription", "medication"],
            replacement="I'm not qualified to give medical advice. Please consult a doctor.",
        ),
        Patter.guardrail(
            name="no_competitor",
            check=lambda text: "competitor" in text.lower(),
            replacement="I can only discuss our own products.",
        ),
    ],
)
```

### `Patter.guardrail()` Parameters

<ParamField body="name" type="str" required>
  Identifier used in log warnings when the guardrail fires.
</ParamField>

<ParamField body="blocked_terms" type="list[str] | None" default="None">
  List of words or phrases. Any case-insensitive match blocks the response.
</ParamField>

<ParamField body="check" type="Callable[[str], bool] | None" default="None">
  Custom callable that returns `True` to block the response. Evaluated after `blocked_terms`.
</ParamField>

<ParamField body="replacement" type="str" default="I'm sorry, I can't respond to that.">
  Text spoken instead of the blocked response.
</ParamField>

## Dynamic Variables

Use `{placeholder}` syntax in your system prompt and provide values at agent creation time.

```python
agent = phone.agent(
    system_prompt="You are {name}'s personal assistant. Today is {date}.",
    variables={
        "name": "Alice",
        "date": "April 3, 2026",
    },
)
```

Variables are substituted into the system prompt before the call starts. This is useful for personalizing agent behavior per call.

## DTMF Input

Keypad presses from the caller are automatically forwarded to the agent as `[DTMF: N]` text in the transcript, where `N` is the key pressed (0-9, *, #).

No additional configuration is needed. The agent can be instructed to handle DTMF via the system prompt:

```python
agent = phone.agent(
    system_prompt="""You are a phone menu system.
When the user presses 1, transfer to sales.
When the user presses 2, transfer to support.
Listen for [DTMF: N] inputs.""",
)
```

## Call Transfer

A `transfer_call` system tool is automatically injected into all agents. When the agent decides to transfer the call, it invokes this tool, which triggers a redirect via the telephony provider's API.

Instruct your agent when to transfer:

```python
agent = phone.agent(
    system_prompt="""You are a receptionist.
If the caller asks for sales, transfer them to +15550009999.
If the caller asks for support, transfer them to +15550008888.""",
)
```

## First Message

The `first_message` parameter makes the agent speak immediately when the call connects, before the caller says anything.

```python
agent = phone.agent(
    system_prompt="You are a helpful assistant.",
    first_message="Hello! Thanks for calling Acme Corp. How can I help?",
)
```

<Note>
An AI disclosure message plays automatically at the start of every call. This is non-optional and precedes the `first_message`.
</Note>

## Conversation History

All callbacks (`on_call_end`, `on_transcript`) receive a `data` dict that includes a `history` field containing the full conversation transcript:

```python
async def on_call_end(data):
    for entry in data.get("history", []):
        print(f"[{entry['role']}] {entry['text']} ({entry['timestamp']})")

await phone.serve(agent, on_call_end=on_call_end)
```

Each history entry has:
- `role` — `"user"` or `"assistant"`
- `text` — the spoken text
- `timestamp` — ISO 8601 timestamp

## Call Recording

Enable recording with a single flag:

```python
await phone.serve(agent, recording=True)
```

Recordings are stored via your telephony provider (Twilio Recordings API or Telnyx equivalent).
